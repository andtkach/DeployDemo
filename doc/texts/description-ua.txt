ОПИС
-----
Дізнайся про навички деплойменту, які повинен мати кожен розробник — без потреби розраховувати на DevOps-спеціаліста!

У цьому демо ми розглядаємо три стратегії деплойменту в Kubernetes на прикладі реального розподіленого застосунку: React UI, REST API на Express.js, фоновий обробник на Node.js та MongoDB.

Починаємо з запуску всього локально через Docker/Podman, потім налаштовуємо локальний Kubernetes-кластер за допомогою Kind і демонструємо кожну стратегію крок за кроком з моніторингом трафіку в реальному часі.

Що розглядається:
- Запуск розподіленого застосунку локально (API + UI + MongoDB + фоновий процесор)
- Docker Compose для інтеграційного тестування
- Налаштування локального Kubernetes-кластера з Kind та MetalLB
- Rolling Update — оновлення API з v1 до v2 без простою
- Canary deployment — направлення 25% трафіку на нову версію API
- Blue-Green deployment — миттєве перемикання трафіку між версіями UI без простою

Ідеально для розробників, які хочуть зрозуміти, як деплоймент реально працює в продакшені, а не тільки в теорії.

Стек: Node.js, React, MongoDB, Docker, Podman, Kubernetes, Kind, MetalLB, kubectl
